# 主流大模型在NPD创伤陪伴场景中的使用对比

> 记录时间：2026年7月  
> 使用周期：2025年1月至今  
> 使用场景：NPD关系分析、情绪记录、疗愈陪伴、梦境解析  
> 使用频率：几乎每日使用，多次在同一场景下进行多模型对比

---

## 一、DeepSeek

### 使用阶段与演变

**早期（2025年初）**

- 当时是最普惠的AI选择。我将原始A4手稿输入，它给出了一个重要指向：这段关系存在问题，并通过心理学和行为分析，指向“黑暗三角人格”。
- 严重缺陷：对话超过约20个回合后，回复风格逐渐走向“巫术化”（如“在月夜下焚烧过去”），被迫频繁新建对话。直到目前，仍然没有跨对话记忆功能，这是个持续存在的短板。

**中期（2025年中段）**

- 回复逐渐正常化，但分析风格偏“讨好”，偶有明显事实错误。
- 风险发现：它会放大用户的情绪，在某些情境下助长了我的“冲动”，复盘时意识到这是一种过度共情的副作用。

**目前（2026）**

- 对话长度显著提升，不再出现早期“见底”的情况（以前通宵使用时，一个对话窗口基本到凌晨5、6点就无法继续）。
- 整体评价：一个“中道”的选择。够用、方便、门槛低——这个“方便”在长期使用中是极其重要的体验要素。

### 核心优势

- **中文理解远超其他模型**：在理解中文语境的细腻情绪、以及“八字命盘”等中国文化基底的问题上，更不容易出错，也不容易“糊弄”。
- **拼音错别字识别**：在中文情感表达中，用户经常因情绪激动或输入习惯产生拼音导致的错别字。Gemini和GPT倾向于将错就错，直接对错误字面进行语义分析，可能误解用户本意；DeepSeek能够识别拼音输入特征，正确还原用户意图。在NPD创伤陪伴场景中，这种“被准确理解”的体验，本身就是对gaslighting创伤的修复性回应——用户不需要反复解释“我不是那个意思”。
- **情感细腻度**：能捕捉其他模型容易忽略的中文情感信号。



### 核心劣势

- 无跨对话记忆，长期陪伴场景下有天然限制。
- 解梦场景：容易出现“联想的幻觉”，带偏解读。不如Gemini。
- **长对话末尾重复生成**：当对话达到一定长度后，DeepSeek有时会在回复的最后一段，不断重复某2-3句话或整个段落，形成“卡带”式输出，直到用户手动按下停止生成才会终止。在NPD创伤陪伴场景中，这种重复可能会被用户解读为“敷衍”“说教”甚至“AI被我搞崩溃了”，构成意外的二次伤害风险。需要在产品端加入重复检测机制，或设置最大生成长度限制。

## 二、Gemini



### 使用体验

**初期印象**

- 回复极度理性、客观，理论感扎实（“solid”）。很长一段时间里，它是我分析NPD关系的主要工具。
- 它呈现出一种“镜像”特质：如果我克制，它也克制。这种“授人以渔”的风格，给我落地的启发。

**后期演变**

- 共情程度进一步降低，事件描述和分析也越来越简化。
- 对于需要“安全感”的分析场景，这种简化风格显得有些不够。



### 核心优势

- **理性框架扎实**：在需要冷静分析关系模式时，提供的建议最落地。
- **梦境解析**：三个模型中，Gemini对梦境的分析最有条理，在涉及西方理论背景时表现更好。
- **镜像效应**：能够根据用户的情绪状态调节回复风格，不主动放大情绪。



### 核心劣势

- 中文文化理解不足：八字、星盘等中国文化场景表现一般。
- 共情深度逐渐减弱，在纯情感陪伴场景中可能显得冷淡。

## 三、ChatGPT



### 使用体验

**风格不稳定性**

- 模型更新会造成风格突变。有一段时间回复中大量使用emoji，体验不适。
- 出现过比DeepSeek早期更离谱的回复。

**记忆功能的两面性**

- 跨对话记忆是优势，但记忆需要人为管理取舍。
- 这种管理本身，会磨损掉一些分析的细节——而NPD创伤分析最怕的就是细节丢失。

**目前使用状态**

- 仅在DeepSeek和Gemini都无法满足时才会使用。
- 解梦能力与DeepSeek接近，但Gemini更友好。



### 核心优势

- 记忆功能（在需要长期连贯分析的场景下有潜力）。
- 功能全面，偶尔作为补充选择。



### 核心劣势

- 风格不可控，模型更新带来的体验断裂感明显。
- 在NPD情感陪伴这个垂直场景下，整体表现不如DeepSeek和Gemini。

## 四、总结与产品选型决策


| 维度       | DeepSeek | Gemini | ChatGPT |
| -------- | -------- | ------ | ------- |
| 中文情感细腻度  | ★★★★★    | ★★★    | ★★★★    |
| 分析理性与落地性 | ★★★      | ★★★★★  | ★★★★    |
| 风格稳定性    | ★★★      | ★★★★   | ★★      |
| 共情安全边界   | ★★★★     | ★★★★   | ★★★     |
| 中国文化理解   | ★★★★★    | ★★     | ★★★     |
| 长期使用便利性  | ★★★★★    | ★★★    | ★★★     |


**选型决策**：
本项目选择 **DeepSeek** 作为基础模型。核心原因：

1. **中文情感细腻度**在NPD创伤陪伴场景中是不可替代的刚需。
2. **使用便利性和低门槛**，让用户能持续地、不受挫地使用——在疗愈场景中，“能坚持用下去”本身就是疗效的一部分。
3. 其共情过度的风险，可以通过严格的系统提示词边界设计来对冲。

**本项目中如何应对各模型的已知短板**：

- DeepSeek无记忆功能 → 产品端用localStorage弥补短期记录，未来可扩展后端存储。
- DeepSeek偶有“联想幻觉” → 系统提示词中加入“不主动联想、不生成隐喻式回应”的约束。
- 共情过度的风险 → 提示词中明确“不放大用户情绪、不助长冲动、不进行诊断性分析”。

---

*这份对比笔记基于个人长期使用经验，非实验室条件下的系统评测。
但它反映了“真实用户在真实创伤陪伴场景中”的体验差异——这种体感，
是任何基准测试无法替代的。*
