# 产品设计原则与专业依据

本文档列出了“事实锚点记录器”的核心设计原则，以及每条原则背后对应的心理学、伦理学或法律依据。这些原则不是在设计前翻书找的——它们来自设计者亲身经历的创伤疗愈过程，事后才找到专业理论的印证。

---

## 设计原则与依据对照表

| 设计原则 | 背后的专业依据 |
| --- | --- |
| **不诊断** | 中国官方依据 ICD-10/11，美国依据 DSM-5。人格障碍诊断需执业精神科医师经多次面诊，观察长期、跨情境的稳定行为模式。AI 和记录工具没有诊断资格。 |
| **不贴标签** | 标签效应（Labeling Theory）：一旦被贴上“NPD”、“讨好型人格”等标签，会固化用户对自我或他人的认知，削弱其看到复杂性和变化的能力。贴标签也是诊断行为的一部分，同样需要执业资格。 |
| **不过度共情** | 人本主义心理学中的“无条件积极关注”不等于“纵容”。过度共情会强化用户的受害者叙事，阻碍其看到自身的力量和选择权。这是心理咨询中的常见伦理边界。 |
| **不给建议** | 心理咨询伦理中的“来访者自主权”：治疗师不应替来访者做决定。给建议会让用户从“依赖他人”转向“依赖AI权威”，没有真正建立起自我判断力。 |
| **引导用户成为自己的记录者** | 叙事疗法核心技术（外化对话、重写对话、见证）与认知行为疗法（CBT）五因素模型的产品化应用。你不是替用户定义问题，而是帮ta看到自己生命中被忽略的“独特结果”。 |
| **用户拥有绝对数据主权** | 数据隐私伦理与心理咨询保密原则：创伤记录是高度敏感的个人信息。所有数据默认存储在用户本地浏览器，不上传任何服务器。API分析作为可选功能，需用户主动开启且数据脱敏后发送。提供完善的导出功能，用户可随时带走自己的数据。 |
| **保留“记仇”等非临床化语言** | 心理治疗中的“去病理化”：让用户用自己的语言定义自己的行为，而不是被迫使用临床术语。真实比体面更重要。 |

## 补充说明

### 为什么不贴标签

“贴标签”是指对用户或用户描述的对象使用诊断性术语。标签效应理论指出，一旦被贴上标签，个体倾向于按照标签来理解自己和他人，这可能导致：

1. **固化认知**：用户可能因为“我是讨好型”这个标签，而忽略自己在某些场景中其实敢于表达的行为。
2. **忽视复杂性**：人的行为是复杂的、情境化的。标签抹平了这种复杂性。
3. **替代自我探索**：用户得到一个标签后，可能停止追问“我为什么会这样”，而是简单地接受“因为我是XX人格”。

在产品中，“不贴标签”意味着：只呈现模式，不命名模式；只展示数据，不解释数据；让用户自己得出结论。

### 为什么不给建议

建议看上去是善意的，但在情感陪伴场景中存在三个风险：

1. **替代用户决策**：替用户做决定，短期让ta感到轻松，长期削弱ta的决策能力。
2. **忽视用户独特性**：每个用户的处境、资源、价值观都不同，通用的建议往往不适用。
3. **制造新的依赖**：用户从依赖他人，转向依赖AI的建议。向外求的模式没有改变，只是换了一个对象。
    
    在产品中，“不给建议”意味着：不写“你应该离开他”、“你需要设立边界”等指导性语句。只帮用户看清ta自己写下的东西，让ta自己决定下一步。
